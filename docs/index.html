Install

One command. No clone. No compile.

Prebuilt binaries for Linux, Windows, and macOS. SHA-256 sums signed with sigstore.

curl -fsSL https://shatilkhan.github.io/knightingale/install.sh | sh
cargo install knightingale
brew install ShatilKhan/tap/knightingale
scoop install https://raw.githubusercontent.com/ShatilKhan/knightingale/main/packaging/scoop/knightingale.json

Then:

knightingale setup && knightingale toggle
Why

A daemon, not a desktop app.

15–30 MB binary, 10–40 MB resident memory. Hits your hotkey, captures the mic, transcribes, types. That's it.

∞

Bring your own key

Eight providers share one HTTP shape: groq, openai, deepinfra, fireworks, lemonfox, sambanova, azure, and any custom OpenAI-compatible server.

⌖

Or run it locally

whisper.cpp via whisper-rs with vulkan, cuda, and metal feature flags. The distil-small.en default fits 4 GB VRAM laptops; large-v3-turbo on RTX-class is real-time.

⊟

Tiny

Single static Rust binary. No Electron. No Chromium. No Python. The whole project is ~14k lines including the design system.

⌘

Cross-platform

Linux (X11 + Wayland), Windows, macOS — one binary per arch. Hotkey, audio, and text injection abstracted behind cpal, global-hotkey, and enigo.

⊘

No telemetry

Zero outbound calls besides the STT endpoint you configured. No analytics, no crash reports, no phone-home. Grep the source for reqwest — there's exactly one HTTP client.

♢

Smart defaults

knightingale model recommend reads your CPU, RAM, and GPU then picks the right Whisper checkpoint for the hardware it's running on.

Privacy

Your audio doesn't leave the bird.

Knightingale does not collect or transmit telemetry of any kind. No usage metrics, no error reports, no analytics, no phone-home. Your audio, transcripts, and configuration stay on your machine unless you explicitly point Knightingale at a cloud provider — in which case audio is sent only to the API endpoint configured in your .env, with your key, and nowhere else.
